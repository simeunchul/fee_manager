# 회비관리 프로그램

팀/모임의 월 회비를 자동으로 관리해주는 데스크톱 프로그램입니다.

- 은행에서 받은 거래내역 엑셀을 끌어다 놓으면, 누가 회비를 냈고 누가 안 냈는지 자동으로 정리
- 입금(회비)과 출금(비용)을 분리해서 한눈에 표시
- 회원 명단도 직접 추가/수정 가능

> 컴퓨터를 잘 다루지 못해도 5분 안에 시작할 수 있도록 만들었습니다.

---

## 처음 설치하기 (1회만)

### 1단계. 파이썬 설치

1. https://www.python.org/downloads/ 접속
2. 노란색 큰 버튼 **Download Python 3.11.x** 클릭 (3.11 이상 권장)
3. 다운받은 파일 실행
4. 설치 화면에서 **반드시 맨 아래 "Add python.exe to PATH" 체크박스를 켜고** 설치
5. "Install Now" 클릭 → 완료될 때까지 대기

### 2단계. 프로그램 다운로드

이 폴더(`fee_manager/`)를 통째로 본인 컴퓨터에 복사하세요.
(예: `바탕화면\fee_manager` 또는 `문서\fee_manager`)

### 3단계. 설치 스크립트 실행

`fee_manager` 폴더 안의 **`install.bat`** 파일을 더블클릭하세요.
- 검정 창이 뜨고 자동으로 필요한 것들을 설치합니다 (2~3분 소요)
- "설치 완료!" 메시지가 보이면 성공
- 만약 "Python이 설치되어 있지 않습니다" 라고 뜨면 1단계로 돌아가서 PATH 체크 여부 확인

## 매번 사용하기

### 프로그램 실행

`fee_manager` 폴더 안의 **`run.bat`** 파일을 더블클릭하세요.
- 검정 창이 뜨고 잠시 후 브라우저가 자동으로 열립니다
- 브라우저에 회비관리 화면이 표시됩니다
- 사용 중에는 검정 창을 닫지 마세요 (창을 닫으면 프로그램이 꺼집니다)

### 처음 한 번 해야 할 일

1. 좌측 사이드바의 **회원관리** 클릭 → 회원 명단 입력
2. 좌측 사이드바의 **설정** 클릭 → 월 회비액 설정 (기본 50,000원)

### 매월 회비 정산하기

1. 본인 은행 인터넷뱅킹에 로그인
2. 거래내역조회 → 이번 달 기간 선택 → 엑셀(.xls/.xlsx)로 다운로드
3. **거래내역 업로드** 페이지에 그 파일을 끌어다 놓기
4. 자동으로 회원별 납부 여부가 매칭됩니다
5. **월별 납부현황** 페이지에서 미납자 확인
6. 미납자에게 연락 (회원관리에 등록한 연락처가 표시됨)

---

## 화면 설명

| 페이지 | 설명 |
|---|---|
| **홈** | 이번 달 요약 - 미납자 수, 회비 수입, 비용 지출, 최근 거래 |
| **회원관리** | 회원 추가/수정/삭제, 활성/비활성 토글 (탈퇴자 처리) |
| **거래내역 업로드** | 은행 엑셀 업로드 → 자동 분류 → 미리보기 → 저장 |
| **월별 납부현황** | 회원 × 월 매트릭스 (✓/✗), 회원별 납부 이력 |
| **비용관리** | 출금 내역을 카테고리별로 집계, 카테고리 직접 수정 가능 |
| **설정** | 회비액, 마감일, 비용 카테고리, 저장소 백엔드 |

## 데이터는 어디에 저장되나요?

기본값은 **이 프로그램 폴더 안의 `data/` 폴더에 CSV 파일로** 저장됩니다.
- `data/members.csv` — 회원 명단
- `data/transactions.csv` — 모든 거래내역 (분류 결과 포함)
- `data/settings.json` — 설정값

회비 운영자가 바뀌거나 다른 컴퓨터에서 이어서 쓰고 싶을 때는
**이 `data/` 폴더 전체를 복사해서 옮기면** 됩니다.

> 구글 스프레드시트에 저장하고 여러 사람이 동시에 보고 싶다면
> 별도로 OAuth 설정이 필요합니다. (`credentials/SETUP_OAUTH.md` 참고 — 개발자용)

## 자주 묻는 질문

**Q. 친구가 송금할 때 이름이 회원명단과 살짝 달라요 (예: 회원명 "김철수", 송금 "김철수a")**
A. 프로그램이 어느 정도는 자동으로 맞춰줍니다 (괄호/공백 무시). 그래도 매칭 안 되면 회원관리에서 그 이름으로 더미 회원을 추가하거나, 회원명을 송금 이름과 동일하게 수정한 뒤 거래내역 업로드 페이지의 **회원명단으로 전체 재분류** 버튼을 누르세요.

**Q. 한 사람이 한 달에 회비를 두 번 냈어요 (또는 한 번에 두 달치)**
A. 두 번 입금된 경우는 모두 그 달의 납부로 인정됩니다. 두 달치를 한 번에 낸 경우는 현재 다음 달이 미납으로 표시됩니다 (수동 보정 필요 — 다음 버전에서 보완 예정).

**Q. 은행이 자동 감지가 안 됩니다**
A. 거래내역 업로드 페이지의 "은행 선택" 드롭다운에서 직접 선택하세요. 그래도 안 되면 "기타 (수동 매핑)" 을 선택하세요.

**Q. 컬럼명이 달라서 파싱이 안 돼요**
A. `config/bank_columns.yaml` 파일을 메모장으로 열고 본인 은행의 실제 컬럼명을 추가하세요.

**Q. 이 데이터를 인터넷에 보내나요?**
A. 아니요. 모든 데이터는 본인 컴퓨터의 `data/` 폴더에만 저장됩니다.
   (구글 시트 연동 옵션을 켰을 때만 본인의 구글 계정에 업로드됩니다)

## 폴더 구조

```
fee_manager/
├── README.md              ← 이 문서
├── install.bat            ← 최초 1회 더블클릭
├── run.bat                ← 매번 사용 시 더블클릭
├── requirements.txt
├── app.py                 ← 홈 화면
├── pages/                 ← 각 메뉴 페이지
├── src/                   ← 핵심 로직 (수정 불필요)
├── config/
│   └── bank_columns.yaml  ← 은행별 엑셀 컬럼 매핑
├── data/                  ← 회원/거래/설정 저장 (자동 생성)
└── credentials/
    └── SETUP_OAUTH.md     ← 구글 시트 연동 가이드 (개발자용)
```

## 문제가 생겼을 때

- `run.bat` 검정 창에 빨간색 에러가 뜨면 그 내용을 캡처해서 개발자에게 전달
- `data/` 폴더는 **항상 백업해두세요** (회원 정보가 모두 들어있음)
