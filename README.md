# 회비관리 프로그램

팀/모임의 월 회비를 자동으로 관리해주는 데스크톱 프로그램입니다.

- 은행에서 받은 거래내역 엑셀을 끌어다 놓으면, 누가 회비를 냈고 누가 안 냈는지 자동으로 정리
- 입금(회비) / 출금(비용) 분리, 누적 회비 / 누적 비용 / 잔여금액까지 한 화면에
- 회원·회비액 변경 시 모든 거래 자동 재분류 — 회비액 바꿔도 매트릭스가 즉시 갱신
- 미납자 명단을 엑셀/CSV로 한 클릭에 export → 카톡·이메일 첨부

> 컴퓨터를 잘 다루지 못해도 **install.bat 더블클릭 한 번**으로 설치되도록 만들었습니다.
> 별도로 Python을 깔 필요 없습니다 — 프로그램 안에 다 들어 있어요.

---

## 처음 설치하기 (1회만)

### 1단계. 프로그램 다운로드

GitHub Releases 페이지에서 최신 release의 두 파일을 받습니다:

- `fee_manager_portable_X.Y.Z.zip` (90MB대)
- `install.bat` (2KB)

두 파일을 **같은 폴더**(예: 바탕화면)에 두세요.

> Source code (zip/tar.gz) 는 받지 마세요 — 코드 archive 용입니다.

### 2단계. 설치

`install.bat` 더블클릭. 자동으로:

1. `C:\Users\<사용자>\fee_manager\` 폴더에 압축이 풀림
2. 바탕화면에 **회비관리** 바로가기 생성

"설치 완료!" 메시지가 보이면 끝.

## 매번 사용하기

바탕화면의 **회비관리** 바로가기를 더블클릭하세요.

- 검정 cmd 창이 뜨고 잠시 후 브라우저가 자동으로 열립니다
- 사용 중에는 검정 창을 닫지 마세요 (창을 닫으면 프로그램이 꺼집니다)
- 종료하려면 검정 창에서 Ctrl+C 두 번

### 처음 한 번 해야 할 일

1. 좌측 사이드바 **회원관리** → 회원 명단 입력
2. 좌측 사이드바 **설정** → 월 회비액 설정 (기본 50,000원)

### 매월 회비 정산하기

1. 본인 은행 인터넷뱅킹에서 거래내역조회 → 그 달 기간 선택 → 엑셀(.xls/.xlsx)로 다운로드
2. **거래내역 업로드** 페이지에 파일을 끌어다 놓기
3. 자동으로 회원별 납부 여부가 매칭됩니다
4. **월별 납부현황** 페이지에서 미납자 확인 (해당 월 미납자 엑셀 다운로드도 한 클릭)
5. 미납자에게 연락 (회원관리에 등록한 연락처가 표시됨)

## 화면 설명

| 페이지 | 설명 |
|---|---|
| **홈** | 이번 달 요약 + 누적 요약(회비/비용/잔여금액) + 월별 차트 + 최근 거래 |
| **회원관리** | 회원 추가/수정/삭제, 활성/비활성 토글 (탈퇴자 처리) |
| **거래내역 업로드** | 은행 엑셀 업로드 → 자동 분류 → 미리보기 → 저장. 저장된 거래의 개별 삭제·수동 매칭 변경도 같은 페이지에서 가능 |
| **월별 납부현황** | 회원 × 월 매트릭스 (✓/✗), 미납자 명단 엑셀/CSV 다운로드, 회원별 납부 이력 |
| **비용관리** | 출금 내역을 카테고리별로 집계, 카테고리 직접 수정 가능 |
| **설정** | 회비액, 마감일, 비용 카테고리 |

## 데이터는 어디에 저장되나요?

본인 컴퓨터의 사용자 폴더 안 `.fee_manager/` 에 저장됩니다 (다른 사람과 공유 안 됨).

```
C:\Users\<사용자>\.fee_manager\
├── members.csv         ← 회원 명단
├── transactions.csv    ← 모든 거래내역 (분류 결과 포함)
└── settings.json       ← 설정값
```

> 폴더 이름이 점(`.`)으로 시작합니다. 탐색기 주소창에 `%USERPROFILE%\.fee_manager` 입력해서 들어갈 수 있습니다.

**왜 프로그램 폴더 밖인가**: 자동 업데이트로 프로그램 폴더가 통째로 갈아끼워져도 데이터가 안 날아가도록.

**다른 PC로 이전**: 위 3개 파일을 복사해서 새 PC의 같은 위치에 붙여넣으세요.

**팀원 공유**: 월별 납부현황 페이지의 **엑셀 다운로드** 버튼으로 미납자/요약을 내려받아 카톡·이메일로 전달.

## 자동 업데이트

새 버전이 GitHub Releases에 올라오면 다음 실행 시 사이드바에 **새 버전 X.Y.Z** 배너가 뜹니다.

1. **업데이트 받기** 버튼 클릭
2. 자동으로 다운로드 → streamlit이 종료되었다가 새 버전으로 다시 시작
3. 브라우저 새로고침

데이터는 그대로 보존됩니다. 사용자가 따로 할 일은 클릭 한 번뿐.

## 자주 묻는 질문

**Q. 친구가 송금할 때 이름이 회원명단과 살짝 달라요 (예: 회원명 "김철수", 송금 "김철수a")**
A. 프로그램이 괄호/공백은 자동으로 무시합니다. 그래도 매칭 안 되면:
- 거래내역 업로드 페이지의 **저장된 거래내역 전체 보기 → 수정/삭제** 에서 그 거래의 회원매칭을 직접 지정 → 변경사항 저장
- 한 번 직접 매칭하면 다음부터 자동 재분류에서도 그 매칭이 보존됩니다

**Q. 같은 거래내역 파일을 실수로 두 번 올렸어요**
A. 자동으로 중복 거래는 무시됩니다. 같은 (일시 + 상대방 + 금액 + 적요) 거래는 한 번만 저장돼요. 업로드 결과에 "X건 저장 (중복 Y건 무시)"로 표시됩니다.

**Q. 한 사람이 한 달에 회비를 두 번 냈어요 (또는 한 번에 두 달치)**
A. 두 번 입금된 경우는 모두 그 달의 납부로 인정됩니다. 두 달치를 한 번에 낸 경우는 매트릭스에 다음 달이 미납으로 표시됩니다 — 거래내역 업로드 페이지에서 그 거래의 회원매칭을 다시 지정하거나, 회비액 ± 10% 안 금액이라 자동으로 회비로 잡힙니다.

**Q. 회비액을 중간에 바꿨어요. 기존 거래는 어떻게 되나요?**
A. 설정 페이지에서 저장하는 즉시 모든 기존 거래가 새 회비액 기준으로 자동 재분류됩니다. "거래 N건의 분류를 자동 갱신했습니다" 안내가 뜹니다.

**Q. 은행이 자동 감지가 안 됩니다**
A. 거래내역 업로드 페이지의 "은행 선택" 드롭다운에서 직접 선택하세요. 그래도 안 되면 "기타 (수동 매핑)" 을 선택.

**Q. 컬럼명이 달라서 파싱이 안 돼요**
A. `config/bank_columns.yaml` 파일을 메모장으로 열고 본인 은행의 실제 컬럼명을 후보에 추가하세요. 단 portable로 설치한 경우 `C:\Users\<사용자>\fee_manager\config\bank_columns.yaml`.

**Q. 이 데이터를 인터넷에 보내나요?**
A. 아니요. 모든 데이터는 본인 컴퓨터의 `.fee_manager/` 폴더에만 저장됩니다. 자동 업데이트 체크 시에만 GitHub API로 새 버전 정보를 조회합니다 (개인 정보 전송 없음).

**Q. 백신(AhnLab 등)이 차단해요**
A. cryptography 등 일부 OSS 패키지의 정상 .pyd 파일이 행동 기반 탐지에 false positive로 잡히던 이슈는 1.0.0+ 버전에서 해당 의존성을 제거해 해결됐습니다. 그래도 차단 메시지가 뜨면 백신에서 `_runtime\` 폴더를 예외 처리하세요.

## 개발자용 — 빌드와 배포

본인이 코드를 수정하고 새 버전을 배포하는 경우.

### 처음 빌드 환경 셋업

Windows 10/11 64bit + Long Path Support 활성화 필요. 관리자 PowerShell에서 한 번만:

```powershell
New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" `
  -Name "LongPathsEnabled" -Value 1 -PropertyType DWORD -Force
```

### dev 모드로 실행 (코드 수정 즉시 반영)

`dev_run.bat` 더블클릭 → `http://localhost:8503` 접속.
portable과 포트 분리(8503)라 동시 실행 가능.

### 새 버전 빌드 + 릴리스

```powershell
# 1) 빌드 (VERSION 자동 증가)
powershell -ExecutionPolicy Bypass -File build_portable.ps1 -Bump patch
# -Bump patch | minor | major  중 하나

# 2) git commit
git add .
git commit -m "release v1.0.X"
git push

# 3) GitHub Releases UI 에서:
#    - "Draft a new release" → tag v1.0.X (새 tag로 자동 생성)
#    - dist/fee_manager_portable_1.0.X.zip + dist/install.bat 첨부
#    - Publish
```

빌드는 처음 한 번만 ~10분 (임베디드 Python 다운로드), 이후 ~1분 (`_runtime/` 캐시).
사용자 PC에선 다음 실행 시 사이드바 배너로 자동 안내.
